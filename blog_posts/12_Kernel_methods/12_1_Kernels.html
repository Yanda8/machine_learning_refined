12.1.1 Additional examples of kernel universal approximators¶

What generally characterizes kernel-based universal approximators are their lack of internal parameters and straightforward organization - with very often the units of a particular family of kernel approximators being organized in order of increasing capacity. These characteristics made many kernel approximators, like the polynomials we have seen previously as well as the sinusoidal and Fourier examples given below, extremely popular in areas adjacent to machine learning for many centuries, e.g., mathematics, physics, and engineering.

Example 1. Sinusoidal kernel approximators¶

The collection of sine waves of increasing frequency is a common example of a classic kernel approximator. This set of nonlinear features consists of sine waves with ever increasing frequency by an integer factor

\begin{equation} f_1(x) = \text{sin}(x), ~~ f_2(x) = \text{sin}(2x), ~~ f_3(x) = \text{sin}(3x), ... \end{equation}

where the $m^{th}$ element given as $f_m(x) = \text{sin}(mx)$. Below we plot the table of values for the first four of these catalog functions using their equations.

As with the polynomials, notice how each of these catalog of elements if fixed and that the elements are ordered in terms of their capacity (from low to high). Again, they have no tunable parameters inside, the third element always looks like $f_3(x) = \text{sin}(x)$ - that is it always takes on that shape. Also note, like polynomials to generalize this catalog of functions to higher dimensional input we shove each coordinate through the single dimensional version of the function separately. So in the case of $N=2$ inputs the functions take the form

\begin{equation} f_1(x_1,x_2) = \text{sin}(x1), ~~ f_1(x_1,x_2) = \text{sin}(2x_1)\text{sin}(5x_2), ~~ f_3(x_1,x_2) = \text{sin}(4x_1)\text{sin}(2x_2), ~~ f_4(x_1,x_2) = \text{sin}(7x_1)\text{sin}(x_2), ~~ ... \end{equation}

And these are listed in no particular order, and in general we can write a catalog element as $f_m(x_1,x_2) = \text{sin}(px_1)\text{sin}(qx_2) $ where $p$ and $q$ are any nonnegative integers.

In general for $N$ dimensional input a sinusoidal unit looks like

\begin{equation} f_m\left(\mathbf{x}\right) = \text{sin}\left(m_1x_1\right)\text{sin}\left(m_2x_2\right)\cdots \text{sin}\left(m_Nx_N\right) \end{equation}

where $m_1,\,m_2,\,...,m_N$ are nonnegative integers.

Example 2. Fourier bases¶

Similar to the sinusoid example above the Fourier basis - so named after its inventor Joseph Fourier who first used these functions in the early 1800s to study heat diffusion - consists of the set of period $1$ sine and cosine waves with ever increasing frequency of the form

\begin{equation} \begin{cases} f_{2m-1}\left(x\right)=\mbox{sin}\left(2 \pi mx\right) & \,\,\,\textrm{for all }m\geq1\\ \,\,\,\, f_{2m}\left(x\right)=\mbox{cos}\left(2 \pi mx\right) & \,\,\,\textrm{for all }m\geq1. \end{cases} \end{equation}

It is also common to write the Fourier basis approximation using classic complex exponential definitions of both cosine and sine, i.e., $\mbox{cos}\left(\alpha\right)=\frac{1}{2}\left(e^{i\alpha}+e^{-i\alpha}\right)$ and $\mbox{sin}\left(\alpha\right)=\frac{1}{2i}\left(e^{i\alpha}-e^{-i\alpha}\right)$. With these complex exponential definitions, one can show that with a scalar input $x$, Fourier basis elements can be written in complex exponential form $f_{m}\left(x\right)=e^{2\pi imx}$

For a general $N$ dimensional input each polynomial feature takes the analogous form $f_{m}\left(\mathbf{x}\right)=x_{1}^{m_{1}}x_{2}^{m_{2}}\cdots x_{N}^{m_{N}}$. Likewise, using the complex exponential notation each multidimensional Fourier basis element takes the form $f_{m}\left(\mathbf{x}\right)=e^{2\pi im_{1}x_{1}}e^{2\pi im_{2}x_{2}}\cdots e^{2\pi im_{N}x_{N}}$ where $m_1,\,m_2,\,...,m_N$ are nonnegative integers.

The challenge of scaling kernels with large input¶

When using polynomial units very often - as we saw in Chapter 11 - we used complete packages of monomials as a polynomial of a certain degree. For example, a polynomial model of degree $D$ when $N=2$ consists of all monomial units of the $f_m\left(\mathbf{x}\right) = x_1^px_2^q$ where $p + q \leq D$. More generally with $N$ dimensional input a polynomial unit takes the form

\begin{equation} f_m\left(\mathbf{x}\right) = x_1^{m_1}x_2^{m_2}\cdots x_N^{m_N} \end{equation}

and a to construct a polynomial model of degree $D$ we collect all such terms where $m_1 + m_2 + \cdots + m_N \leq D$. In other words, unless used in tandem with boosting (see Section 11.3) we virtually always use polynomials units as a complete package of units, not individually. One reason for doing this is that since the polynomial units are ordered from decreasing to increasing capacity - when including a unit of a particular capacity it makes some sense organizationally speaking to include all other units in the family having lesser capacities. Packaging polynomial units in this way is not something that we must do when employing them, but is a sensible and common practice.

Like polynomials, it is also common when employing sinusoidal and Fourier units to use them in organized, similar way - that is as 'complete packages' - since they too are ordered in terms of their individual capacities from low to high. For example, in analogy to a $D$ polynomial, we often package a 'degree $D$' sinusoid consisting of all units like those in the example above where $\text{max}\left(m_1,\,m_2,\,...,m_N\right) \leq D$. This choice of packaging is largely a convention (that is - a popular way of forming the collection), and we could also package units precisely as we do with the corresponding polynomial models. It is likewise commonplace when using Fourier units to employ them in complete packages as well. A 'degree $D$ Fourier model' consists - similar to the sinusoidal - of all units of the exponential form given in the example above where $\text{max}\left(m_1,\,m_2,\,...,m_N\right) \leq D$.

However a very serious practical issue presents itself when employing kernels like these when used in complete packages of units like the degree $D$ polynomial: even with a moderate sized input dimension $N$, the corresponding dimension $M$ of the package of transformed features (the number of units in the package) grows rapidly with $N$ and quickly becomes prohibitively large in terms of storage and computation. In other words, the number of units of a typical kernel approximator suffers from the curse of dimensionality, growing exponentially with the size of input. For example:

  • The precise number $M$ of non- bias features/feature weights of a degree $D$ polynomial of an input with dimension $N$ is $\left(\begin{array}{c} N+D\\ D \end{array}\right)-1=\frac{\left(N+D\right)!}{N!D!}-1$. Even if the input dimension is of reasonably small size, for instance $N=100$ or $N=500$, then just the associated degree $D=5$ polynomial feature map of these input dimensions has dimension $M= 96,560,645$ and $M=268,318,178,226$ respectively! In the latter case we cannot even hold the feature vectors in memory on a modern computer.
  • The corresponding number of transformed features with a package of degree $D$ Fourier basis elements is even more gargantuan: the degree $D$ Fourier feature collection of arbitrary input dimension $N$ has $\left(2D + 1\right)^N$ associated/feature weights. When $D=5$ and $N=80$ this is $11^{80}$, a number larger than current estimates of the number of atoms in the visible universe!

This serious scaling issue motivates the so-called kernel trick described in the next Section, that extends the use of kernels - and more specifically the use of distinct 'degree $D$' packages of classic kernel units - to problems with high dimensional input.

This cursory analysis also indicates that since the total number of units of a fixed kernel approximator growing exponentially in the input dimension like this, any approach to selecting units for a nonlinear model is problematic in general. For example, if we chose a smaller set of say just those units with the same capacity - e.g., for polynomials this would be all units where $m_1 + m_2 + \cdots + m_N = D$ - we would still end up with a combinatorially large number of units to employ. On the other hand, choosing a small set of units randomly becomes less and less effective since as $N$ increases there are exponentially more units to choose from.
